\doxysubsection{Detailed Description}
Represents log-\/likelihood ratio (LLR) values. 

Log-\/likelihood ratios are represented as {\ttfamily int8\+\_\+t} values between {\ttfamily -\/LLR\+\_\+\+MAX} and {\ttfamily LLR\+\_\+\+MAX}. The values {\ttfamily {$\pm$}LLR\+\_\+\+INFTY = {$\pm$}127} are reserved for denoting fixed bits, that is bits that are surely set to zero (log-\/likelihood ratio equal to {\ttfamily +LLR\+\_\+\+INFTY}) or one (log-\/likelihood ratio equal to {\ttfamily -\/LLR\+\_\+\+INFTY}).

Soft XOR operation. 

The soft XOR between two LLRs {\ttfamily x} and {\ttfamily y} is the log-\/likelihood ratio corresponding to the logical XOR operation $a \oplus b$, where $a$ and $b$ are the bits whose soft representation is {\ttfamily x} and {\ttfamily y}, respectively.

\begin{DoxyRemark}{Remarks}
For complexity reasons, the true soft xor function  \[
\text{soft_xor}(x, y) = \sign(x y) \cdot \Phi\Bigl(\Phi(\abs{x}) +
\Phi(\abs{y})\Bigr) \qquad\text{with }
\Phi: \mathbb{R}_+ \to \mathbb{R}_+; u \mapsto \Phi(u) = -\ln\Bigl(\tanh\Bigl(\frac{u}{2}\Bigr)\Bigr)
\] is replaced by the common approximation  \[
\text{soft_xor}(x, y) \approx \sign(x y) \cdot \min\bigl\{\abs{x}, \abs{y}\bigr\}.
\] 
\end{DoxyRemark}
