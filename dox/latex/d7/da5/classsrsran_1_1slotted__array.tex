\doxysubsection{Detailed Description}
\subsubsection*{template$<$\mbox{\hyperlink{namespacesrsran_a58e7c9f2c51a9ad6a0899b46d5315723}{typename}} T, size\+\_\+t N, \mbox{\hyperlink{classbool}{bool}} Embedded\+Storage = true$>$\newline
class srsran\+::slotted\+\_\+array$<$ T, N, Embedded\+Storage $>$}
Array of optional elements, with the following characteristics\+: 


\begin{DoxyItemize}
\item Index-\/based lookup to elements,
\item The iteration is in order of indexes and correctly skips absent elements,
\item Stable -\/ Pointer/\+References/\+Iterators remain valid throughout the object lifetime.
\item Memory overhead -\/ The container creates an array/vector up-\/front of size {\ttfamily N}, even if its number of elements is still zero.
\item Fragmentation -\/ The container has holes in its internal data storage for every absent indexes. This reduces the efficiency of operations like iteration in scenarios of high fragmentation. The advantages of this container include\+:
\item O(1) complexity for index-\/based lookup, creation, removal (just one array indexing),
\item No allocations during element creation or deletion,
\item Pointer stability of its elements.
\item Index-\/based lookup is thread-\/safe if different threads use different indexes. This container may be unsuitable for scenarios where {\ttfamily N} is large, due to the increased memory overhead and potential fragmentation.

{\em Embedded\+Storage} & Whether the array is stored in the object (via std\+::array) or separately in the heap (via std\+::vector). \\
