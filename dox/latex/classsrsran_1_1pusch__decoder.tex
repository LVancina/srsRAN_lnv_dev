\doxysubsection{Detailed Description}
PUSCH decoder interface. 

Recovers a UL-\/\+SCH transport block from a PUSCH codeword. Reverting the encoding operations described in TS38.\+212 Sections 6.\+2.\+1-\/6.\+2.\+6, the codeword is first split into rate-\/matched codeblocks. Then, each codeblock is restored to its base rate, combined with previous retransmissions, and decoded. Finally, if all blocks pass the CRC check, the data bits from all codeblocks are concatenated to form the UL-\/\+SCH transport block. If applicable, a last transport-\/block CRC is computed and verified.

The PUSCH decoder is configured using the method \doxylink{classsrsran_1_1pusch__decoder_abfc3c42e784ff24d28827bb5973782c3}{new\+\_\+data}. This method returns a \doxylink{classsrsran_1_1pusch__decoder__buffer}{pusch\+\_\+decoder\+\_\+buffer} which is used to write softbits into the decoder.

The decoder could potentially start the code block processing if a codeword length is provided (see set\+\_\+codeword\+\_\+length) prior \doxylink{classsrsran_1_1pusch__decoder__buffer_a80f0303716f871142901de4bd295c2d4}{pusch\+\_\+decoder\+\_\+buffer\+::on\+\_\+end\+\_\+softbits}.
