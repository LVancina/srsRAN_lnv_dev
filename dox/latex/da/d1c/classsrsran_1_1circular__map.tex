\subsubsection*{template$<$\mbox{\hyperlink{namespacesrsran_a58e7c9f2c51a9ad6a0899b46d5315723}{typename}} K, \mbox{\hyperlink{namespacesrsran_a58e7c9f2c51a9ad6a0899b46d5315723}{typename}} T, size\+\_\+t N$>$\newline
class srsran\+::circular\+\_\+map$<$ K, T, N $>$}
Contiguous array of optional objects indexed circularly via a integer key. Random access, insertion and removal should have complexity O(1). Each container element contains 3 fields\+: (i) presence flag, (ii) key, (iii) stored object In case the presence flag is set to true, the stored object exists and its ctor has been called. The following steps are performed to access an element\+:
\begin{DoxyEnumerate}
\item The provided key K is converted to an array index via the operation I = K \% N.
\item The presence flag for the index I is checked. If present,
\item Check if the provided key K corresponds to the key stored in the element. If the keys match,
\item Retrieve the stored object. The container does not resolve collisions, i.\+e. it doesn\textquotesingle{}t allow two objects of different keys but same index to be inserted in the container. It is up to the user to resolve such situations. There is no pointer and iterator invalidation.

{\em T} & type of each element \\
\hline
{\em N} & maximum number of elements \\
