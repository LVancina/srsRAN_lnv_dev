Convolution between two sequences. 

Computes the convolution between two sequences $x$ and $y$, namely $z[n] = \sum_{i} x[i] y[n-i]$ and returns its central part, of the {\itshape same} size as the first input sequence. 
\begin{DoxyTemplParams}{Template Parameters}

\hline
{\em T} & Type of the first input sequence (must be compatible with a span of arithmetic/complex values). \\
\hline
{\em U} & Type of the second input sequence (must be compatible with a span of arithmetic/complex values). \\
\hline
\end{DoxyTemplParams}

\begin{DoxyParams}[1]{Parameters}
\mbox{\texttt{ out}}  & {\em out} & Output sequence -\/ convolution between the input sequences. \\
\hline
\mbox{\texttt{ in}}  & {\em x\+\_\+v} & First input sequence. \\
\hline
\mbox{\texttt{ in}}  & {\em y\+\_\+v} & Second input sequence. \\
\hline
\end{DoxyParams}
\begin{DoxyRemark}{Remarks}
An assertion is raised if the length of the second sequence is larger than the length of the first one. 

This implementation of the convolution is not based on FFT and should only be used when the second input sequence is "{}very"{} short. 
\end{DoxyRemark}
