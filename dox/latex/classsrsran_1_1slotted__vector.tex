\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename T$>$\newline
class srsran\+::slotted\+\_\+vector$<$ T $>$}
Container representing a vector of optional elements. It has the following characteristics\+: 


\begin{DoxyItemize}
\item composed by two vectors, one "{}elements vector"{} for storing the container\textquotesingle{}s elements and one "{}indexing vector"{} to convert external indexes to an index position in the first "{}elements vector"{},
\item Index-\/based lookup of elements, based internally on two vector index lookups,
\item The iteration is in order of indexes and correctly skips absent elements,
\item Unstable -\/ Pointer/\+References/\+Iterators do not remain valid during addition/removal of elements in the container,
\item Memory overhead -\/ Reduced memory overhead compared to {\ttfamily static\+\_\+array$<$\+T, N$>$} because elements are now stored contiguously in memory in the "{}elements vector"{}. However, some memory overhead is expected in the "{}indexing vector"{},
\item Fragmentation -\/ There is only fragmentation in the "{}indexing vector"{}. The advantages of this container include\+:
\item Less memory overhead and fragmentation than {\ttfamily slotted\+\_\+array$<$\+T, N$>$},
\item O(1) complexity for index-\/based lookup, consisting of two vector index accesses,
\item O(1) creation in the best case scenario via push\+\_\+back. O(\+N) if the "{}elements vector"{} needs to be resized,
\item O(1) removal via erase-\/remove pattern. This container may be unsuitable for scenarios where indexes are very scattered, due to the increased memory overhead and fragmentation, or for scenarios where pointer stability is required.
