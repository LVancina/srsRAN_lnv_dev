Bandwidth part indicator -\/ 0, 1 or 2 bits.

Identifies the BWP for the frequency domain allocation. The number of bits is determined by the number of DL BWP configured by higher layers, $n_{\textup{BWP,RRC}}$ (see \doxylink{structsrsran_1_1dci__size__config_ac2c1c34ff3856977b241bf36677832c8}{dci\+\_\+size\+\_\+config\+::nof\+\_\+dl\+\_\+bwp\+\_\+rrc}). It can be either\+:
\begin{DoxyItemize}
\item $\bigl \lceil \log_2(n_{\textup{BWP,RRC}} + 1)\bigr \rceil$ bits if $n_{\textup{BWP,RRC}} \leq 3$, set according to the higher layer parameter {\itshape BWP-\/\+Id} (see TS38.\+331 Section 6.\+3.\+2, Information Element {\itshape BWP-\/\+Id}).
\item $\bigl \lceil \log_2(n_{\textup{BWP,RRC}})\bigr \rceil$ bits otherwise, as per TS38.\+212 Table 7.\+3.\+1.\+1.\+2-\/1.
\end{DoxyItemize}
