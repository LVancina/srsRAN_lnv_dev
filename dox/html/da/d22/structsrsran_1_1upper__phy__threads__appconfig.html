unsigned srsran::upper_phy_threads_appconfig::nof_pusch_decoder_threads = 0

Number of threads for concurrent PUSCH decoding.

If the number of PUSCH decoder threads is greater than zero, the PUSCH decoder will enqueue received soft bits and process them asynchronously. Otherwise, PUSCH decoding will be performed synchronously.

In non-real-time operations (e.g., when using ZeroMQ), setting this parameter to a non-zero value can potentially introduce delays in uplink HARQ feedback.

◆ pdsch_processor_type

std::string srsran::upper_phy_threads_appconfig::pdsch_processor_type = "auto"

PDSCH processor type.

Use of there options:

  • automatic: selects lite implementation if nof_pdsch_threads is one, otherwise concurrent, or
  • generic: for using unoptimized PDSCH processing, or
  • concurrent: for using a processor that processes code blocks in parallel, or
  • lite: for using a memory optimized processor.
