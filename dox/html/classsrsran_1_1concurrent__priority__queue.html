Detailed Description

template<typename T, concurrent_queue_policy... QueuePolicies>
class srsran::concurrent_priority_queue< T, QueuePolicies >

Concurrent priority queue, where the caller specifies the element priority statically while pushing it to the queue.

The prioritization is achieved via multiple queues. The pop functions will always start with the highest priority queue until it is depleted, and then move to the second highest priority queue, and so on.

Template Parameters
TThe type of the pushed/popped object.
QueuePoliciesThe queueing policies of each of the queues of different priorities.
