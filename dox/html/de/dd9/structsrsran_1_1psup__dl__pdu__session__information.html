◆ dl_sending_time_stamp

optional<uint64_t> srsran::psup_dl_pdu_session_information::dl_sending_time_stamp

DL Received Time Stamp. This field indicates the time when the UPF sends the DL PDU Session Information frame with the QMP field set to 1. It is used only in the downlink direction and encoded in the same format as the 64 - bit timestamp format as defined in Section 6 of IETF RFC 5905 [6].

◆ pdu_type

const psup_pdu_type srsran::psup_dl_pdu_session_information::pdu_type = psup_pdu_type::dl_pdu_session_information

PDU Type. The PDU Type indicates the structure of the PDU session UP frame. The field takes the value of the PDU Type it identifies; i.e. "0" for PDU Type 0. The PDU type is in bit 4 to bit 7 in the first octet of the frame.

◆ ppi

optional<psup_ppi> srsran::psup_dl_pdu_session_information::ppi

Paging Policy Indicator (PPI). When present, the Paging Policy Indicator is used for paging policy differentiation (see 3GPP TS 23.501). This field applies to PDU sessions of IP type.

QoS Flow Identifier (QFI). When present this parameter indicates the QoS Flow Identifier of the QoS flow to which the transferred packet belongs.

◆ rqi

bool srsran::psup_dl_pdu_session_information::rqi = false

Reflective QoS Indicator (RQI). This parameter indicates activation of the reflective QoS towards the UE for the transferred packet as described in clause 5.4.1.1. It is used only in the downlink direction. If RQA (Reflective QoS Activation) has not been configured for the involved QoS flow, the RQI shall be ignored by the NG-RAN node.
