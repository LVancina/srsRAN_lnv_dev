Detailed Description

This pool returns spans of bytes, which have been allocated using a ring memory allocator. Differently from ring_buffer_pool, this pool keeps track of previously allocated bytes, and, on each tick, deallocates the memory that it deems to be too old. This extra check gives this pool the ability to detect potential buffer overflows.
