30/*
31 * This file will hold the interfaces and notifiers for the RLC entity.
32 * They follow the following nomenclature:
33 *
34 * rlc_{tx/rx}_{lower/upper}_layer_{[control/data]}_{interface/notifier}
35 *
36 * 1. TX/RX indicates whether the interface is intended for the
37 * TX or RX side of the entity
38 * 2. Lower/Upper indicates whether the interface/notifier interacts
39 * with the upper or lower layers.
40 * 3. Control/Data: indicates whether this interface is necessary for "control"
41 * purposes (e.g., notifying the RRC of protocol error) or "data" purposes
42 * (e.g. handling PDUs).
43 * This distinction is only necessary when interfacing with the upper layers,
44 * and as such, we omit it in the interfaces with the lower layers.
45 * 4. Interface/Notifier: whether this is an interface the RLC entity will
46 * inherit or a notifier that the RLC will keep as a member.
47 *
48 */
