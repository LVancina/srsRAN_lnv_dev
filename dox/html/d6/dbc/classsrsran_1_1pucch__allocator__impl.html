160 // TS 38.213, Section 9.2.3, explains the UE's procedure to multiplex HARQ-ACK reporting of multiple slot and for
161 // different cells.
162 // "The PUCCH resource determination is based on a PUCCH resource indicator field [5, TS 38.212] in a last DCI
163 // format 1_0 or DCI format 1_1, among the DCI formats 1_0 or DCI formats 1_1 that have a value of a
164 // PDSCH-to-HARQ_feedback timing indicator field indicating a same slot for the PUCCH transmission, that the UE
165 // detects and for which the UE transmits corresponding HARQ-ACK information in the PUCCH where, for PUCCH resource
166 // determination, detected DCI formats are first indexed in an ascending order across serving cells indexes for a
167 // same PDCCH monitoring occasion and are then indexed in an ascending order across PDCCH monitoring occasion
168 // indexes". As a result of this, and depending on whether there is any scheduled SRs, the PUCCH allocator can
169 // either allocate a new PUCCH grant or update an existing one by changing the number of HARQ-ACK bits to be
