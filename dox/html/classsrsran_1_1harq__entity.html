Detailed Description

This object handles the DL and UL HARQ processes for each UE's Component Carrier; therefore, for each UE, we instantiate as many harq_entity object as CCs

Creates a HARQ entity for a given UE cell.

Parameters
rntiRNTI of the UE
nof_dl_harq_procsNumber of DL HARQ processes that the UE can support. This value is derived based on the UE capabilities, and passed to the UE via RRC signalling. See TS38.331, "nrofHARQ-ProcessesForPDSCH". Values: {2, 4, 6, 10, 12, 16}.
nof_ul_harq_procsNumber of UL HARQ processes that gNB can support. This value is implementation-defined and can up to 16 (there are up to 4 bits for HARQ-Id signalling).
max_ack_wait_in_slotsDuration in slots before a HARQ process acknowledgement is considered to have gone missing and that the HARQ can be reset.
