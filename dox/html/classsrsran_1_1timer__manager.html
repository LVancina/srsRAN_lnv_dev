Detailed Description

This class implements the service associated with the management of the timers of the application, including the allocation, destruction, starting/halting and timeout triggering of the registered timers. The user can check and control the state of an individual timer via the respective unique_timer handle object.

The timer_manager and unique_timer classes follow a publisher/subscriber model. The timer_manager, the publisher, receives periodic ticks from a clock source (e.g. RF front-end or system clock) that advance its internal clock. It also contains an internal list of subscribed unique_timers. Once enough time has passed and a running timer time-outs, the respective timer gets notified via a callback. The user can control the timer starting time, duration until timeout and timeout callback function definition via the unique_timer interface.

The implementation assumes that the "tick()" method is always called from the same thread and that an unique_timer object is not accessed concurrently by different threads. However, separate unique_timers can be safely used from separate threads. An unique_timer object and the timer_manager communicate with one another via thread-safe queues.
