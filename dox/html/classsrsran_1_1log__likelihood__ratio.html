Detailed Description

Represents log-likelihood ratio (LLR) values.

Log-likelihood ratios are represented as int8_t values between -LLR_MAX and LLR_MAX. The values ±LLR_INFTY = ±127 are reserved for denoting fixed bits, that is bits that are surely set to zero (log-likelihood ratio equal to +LLR_INFTY) or one (log-likelihood ratio equal to -LLR_INFTY).

Saturated sum.

The result is clipped if larger (in absolute value) than log_likelihood_ratio::LLR_MAX. Then, for instance, LLR_MAX + 2 = LLR_MAX and -LLR_MAX - 2 = -LLR_MAX.

[in]range_limitThe input value mapped to LLR_MAX.
Returns
A quantized representation of value as a log_likelihood_ratio quantity.
Note
The quantization in the range (-range_limit, range_limit) is mid-tread uniform, with quantization step range_limit / LLR_MAX .
All values larger (in magnitude) than range_limit, will be clipped and mapped to ±LLR_MAX, depending on their sign.
