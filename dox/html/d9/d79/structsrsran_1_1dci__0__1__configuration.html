Detailed Description

Required parameters for packing a DCI format 0_1 scrambled by C-RNTI, CS-RNTI, SP-CSI-RNTI or MCS-C-RNTI.

Remarks
Defined in TS38.212 Section 7.3.1.1.2.

Member Data Documentation

◆ antenna_ports

unsigned srsran::dci_0_1_configuration::antenna_ports

Antenna ports for PUSCH transmission - 2, 3, 4 or 5 bits.

Indicates the antenna ports the UE must use to transmit the PUSCH and the corresponding DM-RS, as per TS38.212 Section 7.3.1.1.2 and Tables 7.3.1.1.2-6 to 7.3.1.1.2-23, according to wether transform precoding is enabled, the DM-RS type (see dci_size_config::pusch_dmrs_A_type and dci_size_config::pusch_dmrs_B_type) and the maximum DM-RS length (see dci_size_config::pusch_dmrs_A_max_len and dci_size_config::pusch_dmrs_B_max_len).

◆ beta_offset_indicator

optional<unsigned> srsran::dci_0_1_configuration::beta_offset_indicator

Beta offset indicator - 2 bits if present.

Indicates beta offset values as per TS38.213 Table 9.3-3. Required if dynamic beta offsets are configured for UCI on PUSCH, i.e., if dci_size_config::dynamic_beta_offsets is set to true when computing the DCI sizes.

◆ bwp_indicator

optional<unsigned> srsran::dci_0_1_configuration::bwp_indicator

Bandwidth part indicator - 0, 1 or 2 bits.

Identifies the BWP for the frequency domain allocation. The number of bits is determined by the number of UL BWP configured by higher layers, (see dci_size_config::nof_ul_bwp_rrc). It can be either:

  • bits if , set according to the higher layer parameter BWP-Id (see TS38.331 Section 6.3.2, Information Element BWP-Id).
  • bits otherwise, as per TS38.212 Table 7.3.1.1.2-1.

It is required if any BWP besides the initial UL BWP is configured by higher layers, i.e., if dci_size_config::nof_ul_bwp_rrc is not 0.

◆ frequency_resource

unsigned srsran::dci_0_1_configuration::frequency_resource

Frequency domain resource assignment - number of bits as per TS38.212 Section 7.3.1.1.2.

For resource allocation type 0, this field occupies bits, providing the resource allocation as per TS38.214 Section 6.1.2.2.1 (see dci_size_config::nof_ul_rb_groups). For resource allocation type 1, this field takes bits as per TS38.214 Section 6.1.2.2.2. If the resource allocation type is dynamically selected, this field occupies 1 bit more than the resource allocation type with the largest field size. In this case, the MSB bit is used to pack the resource allocation type (see dynamic_pusch_res_allocation_type).

◆ hopping_offset

optional<unsigned> srsran::dci_0_1_configuration::hopping_offset

Frequency hopping offset, selected from the RRC parameter frequencyHoppingOffsetLists.

frequencyHoppingOffsetLists, given by TS38.331 Section 6.3.2, PUSCH-Config Information Element, provides 2 or 4 selectable frequency offsets, as per TS38.214 Section 6.3. The hopping_offset field acts as an index for frequencyHoppingOffsetLists, determining which of the possible offsets is used.

it must be set to one of the following values:

  • (0, 1), if frequencyHoppingOffsetLists has 2 possible offsets.
  • (0, 1, 2, 3), if frequencyHoppingOffsetLists has 4 possible offsets.

It is required if frequency hopping is enabled via the frequency_hopping_flag. Otherwise, leave it unset.

◆ N_ul_hop

optional<unsigned> srsran::dci_0_1_configuration::N_ul_hop

Parameter , as per TS38.212 Section 7.3.1.1.2.

Number of bits used in the DCI payload to pack the frequency hopping offset, represented by hopping_offset.

it must be set to:

  • 1, if the RRC parameter frequencyHoppingOffsetLists has 2 possible offsets.
  • 2, if the RRC parameter frequencyHoppingOffsetLists has 4 possible offsets.

The frequency hopping offset is packed within the most significant bits of the frequency domain resource assignment field, therefore, setting N_ul_hop does not affect the total payload size. It is required if frequency hopping is enabled via the frequency_hopping_flag. Otherwise, leave it unset.

◆ precoding_info_nof_layers

optional<unsigned> srsran::dci_0_1_configuration::precoding_info_nof_layers

Precoding information and number of layers.

The content and actual number of bits is defined by TS38.212 Tables 7.3.1.1.2-2/3/4/5, according to whether transform precoding is enabled or disabled, and the values of the higher layer parameters maxRank (see dci_size_config::max_rank) and codebookSubset (see dci_size_config::cb_subset). It is required for codebook based transmission with more than one antenna port, i.e., if dci_size_config::cb_subset is set when computing the DCI sizes. Otherwise, leave it unset.

◆ ptrs_dmrs_association

optional<unsigned> srsran::dci_0_1_configuration::ptrs_dmrs_association

PT-RS/DM-RS association - 2 bit if present.

This parameter is used to link a PT-RS to a DM-RS. It must be set if PT-RS signals are configured (see dci_size_config::ptrs_uplink_configured) and there is more than one UL DM-RS, that is, when transform precoding is disabled and the maximum configured UL rank is grater than 1 (see dci_size_config::max_rank).

◆ srs_request

unsigned srsran::dci_0_1_configuration::srs_request

SRS request - 2 or 3 bits.

This parameter occupies:

  • 2 bits for a UE in a cell not configured with the higher layer parameter supplementaryUplink (see TS38.331 Section 6.3.2, Information Element ServingCellConfig), filled as per TS38.212 Table 7.3.1.1.2-24.
  • 3 bits for a UE in a cell configured with supplementaryUplink. The first bit is the UL/SUL indicator in TS38.212 Table 7.3.1.1.1-1 and the remaining bits are filled as per TS38.212 Table 7.3.1.1.2-24.

This field may also indicate the associated CSI-RS according to TS38.214 Section 6.1.1.2.

◆ srs_resource_indicator

unsigned srsran::dci_0_1_configuration::srs_resource_indicator

SRS Resource Indicator (SRI).

For codebook based transmission, the SRI selects between one of two SRS resources belonging to different antenna pannels, and occupies 1 bit (see TS38.212 Table 7.3.1.1.2-32). For non-codebook based transmission, the SRI is set as per TS38.212 Tables 7.3.1.1.2-28/29/30/31, and occupies bits (see dci_size_config::pusch_max_layers and dci_size_config::nof_srs_resources).

◆ time_resource

unsigned srsran::dci_0_1_configuration::time_resource

Time domain resource assignment - 0, 1, 2, 3 or 4 bits, as per TS38.214 Section 6.1.2.1.

The number of bits is determined as , where is either:

  • The number of entries in the higher layer parameter pusch-TimeDomainAllocationList (see TS38.331 Section 6.3.2, Information Element PUSCH-Config), if configured.
  • 16 otherwise, which is the number of entries in the default table (see TS38.214 Table 6.1.2.1.1-2).

◆ ul_sch_indicator

unsigned srsran::dci_0_1_configuration::ul_sch_indicator

UL-SCH indicator - 1 bit.

Indicates if UL-SCH should be included in a PUSCH transmission, according to its possible values:

  • 0: Exclude UL-SCH from the PUSCH transmission. Allows UCI on PUSCH without application data.
  • 1: Include UL-SCH in the PUSCH transmission.

◆ ul_sul_indicator

optional<bool> srsran::dci_0_1_configuration::ul_sul_indicator

UL/SUL indicator - 1 bit if present, as per TS38.212 Table 7.3.1.1.1-1.

Set as per TS38.212 Table 7.3.1.1.1-1 to signal a resource allocation on the Supplementary Uplink (SUL) if the UE is configured with the higher layer parameter supplementaryUplink (see TS38.331 Section 6.3.2, Information Element ServingCellConfig) and more than a single carrier in the cell is configured for PUSCH transmission, i.e., if dci_size_config::sul_configured is set to true when computing the DCI sizes. Otherwise, leave it unset.
