bounded_bitset<uci_constants::MAX_NOF_HARQ_BITS> srsran::mac_uci_pdu::pucch_f2_or_f3_or_f4_type::harq_information::payload

Contents of HARQ, excluding any CRC.


Example: If the number of HARQ bits is 20, then it is represented as: [ HARQ_bit_19 ... HARQ_bit_0 ] => [ MSB ... LSB ]. NOTE: If is_valid == true, then the HARQs bits set in payload should be interpreted as an ACK (if the bit is 1) or NACK (if the bit is 0). If is_valid == false, then all HARQs bits set in payload should be interpreted as not-detected; note that these bits cannot be ignored when is_valid == false, as the "not-detected" outcome will be used to acknowledge (negatively) the HARQ processes corresponding to each single bit.
